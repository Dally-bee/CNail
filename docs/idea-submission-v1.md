# CultureNail 初赛 Idea 提交稿 v1

更新：2026-07-13｜初赛截止：2026-08-20｜个人参赛

## 一句话

CultureNail 是面向中国中小跨境卖家的美国文化趋势上新智能体：把带来源和时间的公开趋势证据，转成可进入人工选款、打样和 Amazon US 上架测试的穿戴甲商品概念套装。

## 问题

中小卖家能快速找到视觉素材，却难以回答三个更关键的问题：这个符号在美国语境中意味着什么、它适合哪个具体使用场景、哪些表达会造成文化误读。现有趋势工具往往停在热度展示，设计工具又容易直接生成“好看但无证据、无边界、不可执行”的图片。对于同时出现于万圣节、日常哥特和亡灵节相关语境的骷髅符号，这一断层尤其明显。

## 方案

产品采用五步机会雷达工作流：

1. 机会雷达：读取版本化公开证据，展示来源、时间、市场、指标、局限与置信度。
2. 文化语境：拆分万圣节派对、日常哥特、亡灵节相关纪念语境，并设置敏感性闸门。
3. 概念套装：生成受众、场景、配色、图案、甲型、避免项和小批测试计划。
4. 双边合成研究：用 12 名美国女性消费者模拟角色与 5 名中国跨境卖家模拟角色进行 AI 情景压力测试。
5. Listing 草稿：生成 Amazon US 标题、要点、描述和搜索词假设，并保留证据 ID 与人工复核提示。

输出只承诺“可进入人工选款和打样”，不宣称生产级设计稿、真实用户验证、销量预测或爆款预测。

## 首发切口

- 用户：中国中小跨境卖家
- 品类：穿戴甲
- 市场：美国
- 平台：Amazon US
- 时间：2026 Q4
- 代表符号：骷髅
- 机会形态：证据驱动的机会雷达

首批证据来自 NRF、Pinterest Business、Etsy Seller Handbook、Museum at FIT 与 Smithsonian。平台搜索变化、调查意向和文化资料分开标注，不合成为销量结论。Google Trends 若后续加入，只解释为特定范围内的相对关注度。

## 核心差异

CultureNail 的价值不在“再生成一张图”，而在连接三个通常割裂的环节：

- 证据可追溯：每个 AI 结论必须引用版本化 `EvidenceRecord` ID。
- 文化边界内建：同一符号跨语境时强制分流；亡灵节不是 Mexican Halloween，也不是通用恐怖素材库。
- 双边可执行性：同时检查美国消费者场景假设与中国卖家的成本、供应链、Listing 和小批测试约束。

## 技术实现

Next.js + TypeScript 单页 Demo，服务端 Route Handler 统一处理文化分析、概念、合成评价与 Listing 生成。第一版使用版本化 JSON，不引入数据库。`ModelProvider` 同时支持确定性 mock 和百炼 OpenAI 兼容接口；真实 Provider 仅服务端读取环境变量，7 秒中止，失败即回退到同契约 mock。万相接口只预留且默认禁用。

Demo 预置证据、角色和局部视觉资产，因此在无 Key、无网络、上游超时或页面失效时仍可完成 4 分钟主演示。

## 商业模式

订阅加点数：基础订阅提供机会雷达、证据快照与项目管理；文化分析、批量概念、合成评估和 Listing 生成按点数使用。第一阶段验证重点不是定价最大化，而是卖家是否愿意为“降低文化误判与选款返工”持续使用。

## 可信与安全边界

- 合成研究始终标注 `AI_SIMULATION`，不报告市场比例。
- 不抓取登录态数据，不自动发布或提交 Amazon Listing。
- 不生成无来源的销量、材质、安全、原产地或文化真实性声明。
- 亡灵节概念必须通过人工文化复核；祭坛、真实逝者图像与神圣元素默认禁用。
- 商品视觉仍需版权、商标、制造与平台合规复核。

## 初赛 Demo 成功标准

评委可在 4 分钟内从证据打开一个骷髅机会，看到三语境差异，选择一套日常哥特概念，查看 12+5 模拟压力测试，再生成带证据链的 Amazon US Listing。最后切换/模拟模型失败，流程仍由本地快照完成。

## 当前完成阶段与验证边界

当前完成阶段为“已有 Demo 原型”。第一版已完成本地类型、Lint、测试、构建和差异检查，并可在 mock 模式下离线走通主演示。赛事专属百炼 Key 将在初赛通过后发放；当前尚未获得属于正常状态，不构成阻塞，项目尚未完成真实百炼 API 测试。真实卖家可用性访谈、美国消费者小样本研究、供应链打样、Amazon 合规与版权审查仍待后续真实验证，不能由合成研究替代。
